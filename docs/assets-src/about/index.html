How this site works

A private courier for confidential web documents — you get a link, the link opens the thing, and the moment it shouldn't, it won't.

This page is itself one of those documents, published without a code so anyone can read it. Everything below describes the machinery you just walked through to get here.

The one idea

Someone shares a self-contained HTML document — a report, a dashboard, a deck — with a specific person. Not the whole world, not a login wall, just a link that works for them and can be switched off at any second. That's the entire product. The interesting part is doing it without the usual machinery of accounts, passwords, and sessions.

What happens when you open a share link

A link looks like /a/<slug>?code=…. The slug names an opaque document; the code is a 128-bit secret minted for one recipient. Opening it runs this, in order:

  1. 1
    Check the shape. The slug is validated before touching any database — a malformed one is rejected instantly, giving nothing away.
  2. 2
    Redeem the code, atomically. A single database statement hashes the code, confirms it's valid, unexpired, and unrevoked, and stamps the redemption — all at once, so there's no gap to race.
  3. 3
    Hand back a signed cookie. Success mints a short-lived, signed, HttpOnly cookie scoped to this one document, then redirects to a clean URL with the code stripped out of the address bar.
  4. 4
    Re-check on every single load. The cookie is a convenience, never an authority. Every later request re-asks the database "is this code still good?" — so revoking a code cuts access on the very next click.
  5. 5
    Stream the document from private storage. Only now does the actual content leave the building, straight from a private bucket that has no public address of its own.

The revocation trick. Because authority lives in the database and is re-checked on every request — not baked into a token that's valid until it expires — "turn off this link" is instant and total. No session to wait out, no cache to bust.

Every wrong turn looks identical

An unknown document, a wrong code, an expired code, a revoked code, a link opened in the wrong place — all of them return the exact same page, byte for byte, with the same headers. There's no "that code is expired" versus "no such document" to probe. From the outside, a locked door and a door that was never there are indistinguishable.

What it's built from

The whole thing is one small program running at the edge, close to whoever's reading. It leans on three managed pieces and owns almost no infrastructure:

One edge workerRoutes requests, checks codes, sets cookies, streams documents. No servers to keep alive.
A small databaseThe single source of truth for codes, expiries, and revocations — and the single source of time, so no clock can be gamed.
A private object storeHolds the document bytes. Reachable only through the worker, only after the checks above. It has no public URL.

Publishing is not deploying

Documents are uploaded through the admin panel at runtime — a single file or a whole zipped bundle — and versioned, rolled back, or deleted on the spot. They never live in the source code. That's deliberate: the code that runs this site is public, and none of the confidential content is anywhere near it.

The admin door is a different door

Minting and revoking codes happens behind a corporate single-sign-on gate, and the worker independently re-verifies that identity on its own — it never just trusts the doorman. Recipients never see any of this; they only ever meet a document or the generic page.

Public by choice

Most documents need a code. A few — like this one — are flipped to public: same storage, same worker, same fail-closed plumbing, minus the code check. Flip it back and the gate returns instantly, exactly like a revocation. A public document can also claim a friendly name, which is why this page lives at /about instead of a random slug.

You reached this page with no code — it's a public asset, uploaded through the same panel and served by the same worker as every private one. The difference is one toggle. Everything else on this site is behind a link meant only for you.
